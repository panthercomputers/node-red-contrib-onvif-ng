# Security Policy

## Supported Versions

This project follows a pragmatic support model:

- **Latest published npm version** is supported
- Older versions may receive fixes on a best-effort basis
- Security fixes will be released as patch or minor versions when appropriate

Users are encouraged to stay up to date.

---

## Reporting a Vulnerability

If you discover a security vulnerability, **please do not open a public issue**.

Instead, report it privately using one of the following methods:

- **GitHub Security Advisories** (preferred, if available)
- **Email**: security@panthercomputers.com  
  *(or your preferred contact email)*

Please include:
- A clear description of the issue
- Steps to reproduce (if applicable)
- Affected versions
- Any relevant logs or screenshots

---

## Disclosure Policy

- Vulnerabilities will be acknowledged as quickly as possible
- Fixes will be developed and tested before public disclosure
- Credit will be given to reporters when desired

---

## Scope

This policy applies to:
- The Node-RED ONVIF nodes provided by this package
- Configuration handling and credential storage
- Runtime behavior related to ONVIF device interaction

It does **not** cover:
- Vulnerabilities in third-party dependencies
- Issues caused by misconfigured cameras or networks
- Node-RED core or runtime security

---

Thank you for helping keep this project and its users secure.
